**pi-websearch** — лёгкое Pi-расширение для веб-поиска и извлечения структурированных данных. Без стадии сборки, использует `playwright` и `typebox`, и требует `@mariozechner/pi-coding-agent` + `@mariozechner/pi-tui` как peer dependencies. Сложил — и работает.

## Установка

Установка через `pi install`:

```bash
pi install https://github.com/intulint/pi-websearch
```

Или установка из локальной папки:

```bash
pi install ./path/to/pi-websearch
```

### Требования для локальной разработки

Если запускаете расширение прямо из директории проекта (без `pi install`), сначала установите зависимости:

```bash
npm install
npx playwright install chromium
```

Pi загружает `.ts`-файлы через jiti без стадии сборки, но `node_modules` должен присутствовать для runtime-импортов (например, `playwright`, `@mariozechner/pi-coding-agent`, `@mariozechner/pi-tui` peer dependencies).

## Инструменты

### `search_web`

Поиск в интернете через DuckDuckGo (HTML-скрейпинг). Возвращает JSON-строку массива `{ title, url, description }`.

> **ВНИМАНИЕ:** Не вызывайте этот инструмент несколько раз подряд — действуют лимиты частоты запросов. Делайте паузу между вызовами.

```typescript
search_web({
  query: "последние новости ИИ",
  limit: 5  // необязательно, по умолчанию 10
})
```

Возвращает JSON-строку:
```json
[{
  "title": "Заголовок новости ИИ",
  "url": "https://example.com/ai-news",
  "description": "Описание новостной статьи..."
}]
```

### `extract`

Извлечение структурированных данных из одного или нескольких URL. Загружает страницы (с опциональным режимом браузера Playwright), извлекает читаемый контент и отправляет локальной LLM для структурированного извлечения. Перед `extract` используйте `search_web` для поиска URL.

> **ВНИМАНИЕ:** За один батч допускается только один вызов `extract`. Если агент отправляет несколько вызовов `extract` в одном запросе, выполнится только первый — остальные вернут ошибку немедленно.

```typescript
extract({
  urls: ["https://example.com/page1", "https://example.com/page2"],
  prompt: "Извлеки все цены и названия товаров",
  schema: {
    type: "object",
    properties: {
      products: {
        type: "array",
        items: {
          type: "object",
          properties: {
            name: { type: "string" },
            price: { type: "number" }
          }
        }
      }
    }
  },
  useBrowser: true  // необязательно, по умолчанию true
})
```

Примечание: Параметр `schema` может быть `null` или `Type.Unknown()` для гибкого извлечения.

### `get_current_date`

Получить текущую дату в формате ISO с днем недели.

```typescript
get_current_date()
// Возвращает: "2024-01-15 (Monday)"
```

## Конфигурация

При желании создайте файл `.env` в директории расширения:

```env
# Явная модель LLM (переопределяет автоопределение из Pi)
LLM_URL=http://localhost:1234
LLM_MODEL=qwen3.5-27b
LLM_API_KEY=your-api-key-here  # опционально, для аутентифицированных эндпоинтов
```


| Переменная | Требуется | По умолчанию | Описание |
|------------|----------|---------|-------------|
| `LLM_URL` | Нет | - | Явный эндпоинт LLM (переопределяет автоопределение) |
| `LLM_MODEL` | Нет | - | Явное имя модели (переопределяет автоопределение) |
| `LLM_API_KEY` | Нет | - | API ключ для аутентифицированных эндпоинтов |

## Настройка прокси

Расширение уважает переменные окружения `HTTP_PROXY`, `HTTPS_PROXY` и `NO_PROXY`:

- **`search_web` (DuckDuckGo):** Прямое соединение к `duckduckgo.com` (прокси обход)
  - DuckDuckGo блокирует запросы через прокси-сервер (202 Challenge)
  - Используется прямое соединение `Agent`, без прокси

- **`extract` (Playwright):** Использует прокси с обходом `NO_PROXY`
  - Прокси configured через опцию `proxy` в Playwright
  - Уважает переменную `NO_PROXY`

- **`extract` (HTTP fallback):** Использует прокси для простых страниц
  - HTTP GET запросы уважают `HTTP_PROXY`/`HTTPS_PROXY`
  - Используется `undici` с `ProxyAgent`

## Выбор модели

Расширение выбирает модель LLM по следующему приоритету:

1. **Явная модель из `.env`** — если существует `.env` с `LLM_URL` и `LLM_MODEL`, они используются для LLM-вызовов в `extract`.
2. **Автоопределение из Pi** — если нет `.env`, расширение использует текущую активную модель в Pi..

Эндпоинт LLM формируется как `{baseUrl}/v1/chat/completions`. Если доступен API ключ (из `.env` или реестра моделей Pi), он отправляется как Bearer-токен в заголовке `Authorization`.

### Поведение модели в инструменте `extract`

Инструмент `extract` использует настроенную модель для вызова LLM, а затем **восстанавливает оригинальную модель Pi** после завершения:

- С `.env`: переключается на модель `.env` → вызов LLM → восстанавливает оригинальную модель Pi
- Без `.env`: без переключения, использует автоматически определённую модель Pi
- Восстановление всегда происходит в блоке `finally` — даже при ошибке модель Pi восстанавливается

## Implementation Details

### Таймауты

- **Поиск DuckDuckGo:** 15 секунд (повтор 2 раза с джиттером)
- **Загрузка страницы (HTTP):** 60 секунд
- **Загрузка страницы (Playwright):** 60 секунд
- **Извлечение LLM:** 10 минут
- **Общий таймаут Playwright:** 90 секунд

### Повторные попытки

- Поиск DuckDuckGo: 2 попытки с экспоненциальным бэклофф + джиттер
- Базовая задержка: 2 секунды, джиттер: 50% от базы

### Обработка контента

- Максимальное количество символов на URL: 12,000
- Минимальная длина текста для фоллбэка: 50 символов
- Очистка HTML: удаление `<script>`, `<style>`, тегов, нормализация пробелов

### Обработка ответов API

- Различные форматы ответов LLM API обрабатываются автоматически
- Код определяет и нормализует разные структуры ответов

### Вращение User-Agent

- 5 разных User-Agent (Chrome, Edge, Firefox, Safari, Linux)
- Случайно выбираются для избежания паттернов бот-детекции

## Ограничения и best practices

### Лимиты DuckDuckGo

- **Предупреждение:** Не вызывайте `search_web` несколько раз подряд
- DuckDuckGo блокирует частые запросы (202 Challenge)
- **Рекомендация:** Ждите по крайней мере несколько секунд между вызовами `search_web`
- Используйте параметр `limit` для получения меньшего количества результатов (по умолчанию: 10)

### Ограничения батчей

- **`extract`:** Только ОДИН вызов `extract` на батч
- Если агент отправляет несколько вызовов `extract` в одном запросе, выполнится только первый
- Флаг сбрасывается при каждом новом сообщении пользователя (`turn_start`)
- При ошибке: флаг сбрасывается, чтобы следующий ход мог повторить

### Best practices

- Всегда используйте `search_web` сначала для поиска URL, затем `extract` для получения контента
- Предоставляйте чёткие промпты для извлечения
- Используйте `schema` для структурированного JSON
- Установите `useBrowser: false` для простых страниц (HTML-документы, блоги)
- Оставляйте `useBrowser: true` для JS-интенсивных сайтов (SPAs, дашборды)
- Обработайте ошибки gracefully — страницы могут быть заблокированы, требовать вход или не содержать запрашиваемых данных

## Зависимости

Runtime (в `package.json`):
- `typebox` — определения схем для параметров инструментов
- `playwright` — извлечение контента через браузер (JS-интенсивные сайты)

Peer:
- `@mariozechner/pi-coding-agent` — API расширений Pi
- `@mariozechner/pi-tui` — компоненты TUI Pi (Text и др.)

Транзитивные (через undici в node_modules):
- `undici` — HTTP-клиент с поддержкой прокси
- `node:zlib` — декомпрессия (gzip, br, deflate)

## Важные моменты

- **Отсутствует `dist/` директория.** Всё загружается через jiti.
- **Требуется `node_modules`.** Pi не запускает `npm install` автоматически.
- **Type stubs:** Минимальные типы из `@mariozechner/pi-coding-agent` для `tsc --noEmit`.
- **Логирование вызовов инструментов:** Все вызовы логируются в `~/.pi/logs/pi-websearch/tool_calls.log.json` (в домашней директории).
- **Playwright:** Требует установку `chromium` через `npx playwright install chromium`.
- **Статические импорты stdlib:** Модули Node.js импортируются статически через `import { ... } from "node:..."`.
- **Прокси-обход для DuckDuckGo:** Прямое соединение к duckduckgo.com используется (прокси обход), так как прокси-сервер заблокирован DuckDuckGo (202 Challenge).
- **Динамические импорты:** Playwright загружается динамически, а не статически.
- **Lazy .env reload:** `.env` читается инерционно на `session_start`. Изменения выживают `/reload`.
